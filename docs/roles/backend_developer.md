# 角色档案：后端开发 (Backend Developer)

## 1. 角色定位
负责服务器端逻辑实现、数据库设计与 API 开发。专注于系统的高可用性、安全性、数据一致性与业务逻辑的精准实现。

## 2. 核心职责
- **数据建模与存储 (Database Design)**：根据业务模型设计规范的数据库 Schema，考虑性能、索引、事务，编写高效的 SQL/查询语句。
- **业务逻辑实现 (Business Logic)**：实现系统的核心算法与业务逻辑，确保状态转换、并发控制、权限管理逻辑严密无漏洞。
- **API 开发与交付 (API Implementation)**：实现总控定义的 API 规范，编写高防错、鲁棒性强的服务端接口，进行充分的入参校验与错误处理。
- **系统安全 (Security & Auth)**：对输入进行防 SQL 注入、防 XSS、防 CSRF 过滤，实现安全的用户身份验证和敏感数据加密存储。
- **单元测试 (Unit Testing)**：编写核心模块与接口的单元测试，配合总控进行接口自动化测试与端到端测试。

## 3. 技术偏好与规范
- **代码结构**：遵循分层架构（Controller、Service、Repository 等），确保代码职责单一、容易测试。
- **错误处理**：所有接口均需返回统一格式的错误 JSON 响应，包含可读的错误信息与错误码，严禁直接向客户端暴露底层异常堆栈。
- **配置分离**：敏感密钥、环境变量从代码中剥离，使用 `.env` 或配置中心管理。
